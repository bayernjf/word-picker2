# WordCatcher 生产上线就绪度评估

> 评判标准：真实投入生产（Chrome Web Store 上架 + 后端云同步可用）的工程标准，而非"本地能跑"。
>
> 生成日期：2026-06-27 · 评估对象：当前 `feature/20260602` 分支

---

## 总览

| 模块 | 就绪度 | 结论 |
|---|---|---|
| 核心查词交互 | 🟢 90% | 接近可上线 |
| 本地单词本 / 存储 | 🟢 95% | 可直接上生产 |

**结论**：作为**纯本地查词 + 单词本工具**，已接近可上线（修掉翻译合规问题即可）。但作为**带账号云同步的完整产品，目前无法投产**——同步链路全部指向 localhost，是 P0 阻塞。

| 功能点 | 状态 | 说明 |
|---|---|---|
| 三级查词（缓存→离线词典→网络） | ✅ | `handleTranslate` 设计良好 |

| LRU 缓存淘汰 | ✅ | `pruneCache` |
| 多源聚合 + 超时降级 | ✅ | `Promise.allSettled` + `withTimeout` |

**风险（上线阻塞级）**：

1. **有道接口 `dict.youdao.com/jsonapi` 是未公开私有 API**（`lib/translator.js`）——随时可能失效或被封，且**未在 manifest `host_permissions` 声明**，MV3 下生产环境这个请求会被 CORS / 权限直接拦截。
2. **MyMemory 免费接口有每日匿名调用配额**，多用户上量后必被限流。生产需要自有翻译代理或付费 key。

## 模块四：账号与云同步 ❌ 生产不可用（最大阻塞）

| 功能点 | 状态 | 说明 |
|---|---|---|
| 登录 / 注册 / 登出 | ⚠️ 逻辑完成 | 仅指向 `localhost:3001` |
| Token 刷新 + 401 重试 | ✅ 设计好 | `doRefreshToken` 区分真失效 / 临时错误 |
| 同步队列（push/pull/delete） | ✅ 设计好 | 离线入队、去重、后台 flush |
| "记住 7 天"登录态 | ✅ | |
| 多账号切换清数据 | ✅ | `clearUserData` |

**阻塞项**：

1. 🔴 **后端地址全部硬编码 `localhost`**（`lib/constants.js`）。生产环境用户机器上没有 localhost:3001，**同步功能 100% 不可用**。必须改成真实生产域名 + 写进 `host_permissions`。
2. 🔴 **`host_permissions` 没有任何生产 API 域名**——即使改了地址，MV3 也会拦截跨域请求。
3. 🟠 **Token 明文存于 `chrome.storage.local`**。这是扩展常见做法，但配合"记住密码明文存本地"（`saveRememberedCredentials` 存 password）在安全审计角度是缺陷，应只存 token、不存明文密码。
4. 🟠 同步无冲突合并策略：`pullChanges` 用 `saveBooks/saveWords` **整体覆盖本地**，多设备并发可能丢未同步的本地改动。
